104  // What is it: Coin verification
105  // Who does it: TRANSACTION VERIFIERS
106  // What it does: Verifies the structure of a zerocoin obtained from
107  // a ZEROCOIN_MINT transaction. All coins must be
108  // verified before you operate on them.
109  // Note that this is only part of the transaction
110  // verification process! The client must also check
111  // that (1) the inputs to the transaction are valid
112  // and add up to the value of one zerocoin, (2) that
113  // this particular zerocoin has not been minted before.

122  // Any ZEROCOIN_MINT with an invalid coin should NOT be
123  // accepted as a valid transaction in the block chain.

131  // Who does it: ZEROCOIN CLIENTS & TRANSACTION VERIFIERS
132  // What it does: Collects a number of PublicCoin values drawn from
133  // the block chain and calculates an accumulator.
134  // This accumulator is incrementally computable;
135  // you can stop and serialize it at any point

138  // the same coins can be accumulated in any order

177  // What it does: Create a new transaction that spends a Zerocoin.
178  // The user first authors a transaction specifying
179  // a set of destination addresses (outputs). They
180  // next compute an Accumulator over all coins in the
181  // block chain (see above) and a Witness based on the
